# 🌐 KevinNet DNS

**ابزار جانبی برای [MasterDnsVPN](https://github.com/masterking32/MasterDnsVPN)**

> به صورت خودکار IP‌های ایرانی را اسکن می‌کند تا DNS Resolverهایی که از DPI ایران عبور می‌کنند را پیدا کند — سپس فایل‌های پیکربندی آماده می‌سازد و با یک کلیک وصل می‌شود.

**ساخته شده توسط Kevin Haji · [kevinhaji.com](https://kevinhaji.com) · [kevin.fullstack.dev@gmail.com](mailto:kevin.fullstack.dev@gmail.com)**

## 🚀 نحوه استفاده

### مرحله ۱ — وارد کردن تنظیمات

| فیلد | چی وارد کنید | مثال |
|---|---|---|
| دامنه تانل | ساب‌دامینی که به سرور شما اشاره دارد | `v.example.com` |
| کلید رمزنگاری | کلید ۳۲ کاراکتری از سرور | `4c8da843709463cab27f...` |
| نام کشور / پوشه | نام پوشه خروجی | `Iran` |

### مرحله ۲ — انتخاب تنظیمات اسکن (پیشنهادی برای ایران)

| تنظیم | پیشنهاد برای ایران | توضیح |
|---|---|---|
| **هدف (Target)** | `100` | تعداد Resolverهایی که می‌خواهید پیدا کنید. بیشتر = VPN پایدارتر |
| **همزمانی (Concurrency)** | `80–100` | ایران محدودیت socket دارد — بالاتر از ۱۵۰ نروید |
| **Timeout** | `2–3 ثانیه` | ۳ ثانیه برای شبکه‌های شلوغ ایران ایمن‌تر است |
| **پول (Pool ×1000)** | `200` | ۲۰۰,۰۰۰ IP اسکن می‌شود. بیشتر = Resolver بیشتر |

> **نکته:** اگر Resolver کمی پیدا شد، **Pool** را به `300` یا `500` افزایش دهید.
> **Concurrency را بالای ۱۰۰ نبرید** — باعث خطاهای پنهان در مک و شبکه‌های ایران می‌شود.
> اسکن را ۲-۳ بار اجرا کنید — هر بار IP‌های تصادفی مختلفی تست می‌شود.

### مرحله ۳ — شروع اسکن

روی **▶ شروع اسکن** کلیک کنید. اسکن در ۳ مرحله خودکار اجرا می‌شود:

- **مرحله ۱** — بررسی سریع زنده بودن (~۵-۱۰ دقیقه برای ۲۰۰ هزار IP)
- **مرحله ۲** — امتیازدهی کامل ۶ معیاره: NS→A، TXT، RND، DPI، EDNS، NXD
- **مرحله ۳** — تست واقعی تانل E2E از طریق باینری MasterDnsVPN (فیلتر نهایی)

رنگ‌بندی نتایج:
- 🟢 **★ ۶/۶** — عالی، همه تست‌ها پاس
- 🟡 **◆ ۴-۵/۶** — خوب، احتمالاً کار می‌کند
- 🟠 **▸ ۲-۳/۶** — ضعیف، ممکن است کار کند
- ⚫ **· ۰-۱/۶** — خیلی ضعیف

### مرحله ۴ — ذخیره و اتصال

روی **💾 ذخیره فایل‌ها** کلیک کنید — پوشه‌ای (مثلاً `Iran/`) کنار برنامه ساخته می‌شود:
- `client_config.toml`
- `client_resolvers.txt`
- `MasterDnsVPN` (یا `.exe` در ویندوز)

پروفایل به صورت خودکار در **تب پروفایل‌ها** نیز ذخیره می‌شود — ادامه را بخوانید.

روی **🚀 اتصال MasterDNSVPN** کلیک کنید — ترمینال باز می‌شود و VPN راه‌اندازی می‌شود.

## 📋 تب پروفایل‌ها — ویرایش تنظیمات و مدیریت اسکن‌های ذخیره‌شده

هر بار که بعد از اسکن ذخیره می‌کنید، یک **پروفایل** به صورت خودکار در پوشه `profiles/` کنار برنامه ذخیره می‌شود. بعداً می‌توانید هر پروفایل ذخیره‌شده را — بدون اسکن مجدد — باز کنید، تنظیمات را تغییر دهید و راه‌اندازی کنید.

### نحوه استفاده از تب پروفایل‌ها

۱. روی تب **📋 پروفایل‌ها** در بالای برنامه کلیک کنید
۲. یک پروفایل از لیست سمت چپ انتخاب کنید (نام، تاریخ و تعداد Resolver نشان داده می‌شود)
۳. نام یا هر یک از تنظیمات کلیدی سمت راست را ویرایش کنید
۴. روی **💾 ذخیره تغییرات** کلیک کنید — `client_config.toml` در پوشه کشور فوری بازنویسی می‌شود
۵. روی **🚀 اتصال** کلیک کنید — مستقیم از پروفایل راه‌اندازی می‌شود
۶. روی **🗑 حذف** کلیک کنید — پروفایل (و اختیاراً پوشه خروجی) حذف می‌شود

### 🔧 تنظیمات کلیدی — معنی هر کدام و مقدار بهینه برای ایران

این تنظیمات داخل هر پروفایل هستند و مستقیماً روی عملکرد MasterDnsVPN در شبکه‌های ایران تأثیر می‌گذارند.

---

#### پورت محلی (Listen Port)
**پیش‌فرض:** `18000`
**بهینه برای ایران:** `18000` (اگر برنامه دیگری از این پورت استفاده نمی‌کند، تغییر ندهید)

پورتی که مرورگر یا برنامه شما به آن وصل می‌شود. مثلاً در تنظیمات پروکسی مرورگر، SOCKS5 روی `127.0.0.1:18000` تنظیم کنید. فقط در صورتی تغییر دهید که چیز دیگری پورت ۱۸۰۰۰ را اشغال کرده باشد.

---

#### روش رمزنگاری (Encryption Method)
**گزینه‌ها:** `0 — بدون رمز`، `1 — XOR`، `2 — ChaCha20`، `3 — AES-128-GCM`، `4 — AES-192-GCM`، `5 — AES-256-GCM`
**پیش‌فرض:** `1 — XOR`
**بهینه برای ایران:** `1 — XOR`

محتوای داخل هر پکت DNS را رمزنگاری می‌کند. XOR بهترین انتخاب برای ایران است چون:
- کمترین سربار را داخل پکت‌های کوچک DNS ایجاد می‌کند
- ChaCha20 یا AES قوی‌ترند اما بایت‌هایی اضافه می‌کنند که MTU قابل استفاده را کاهش می‌دهند
- دامنه + کلید تانل یک لایه امنیتی دیگر فراهم می‌کنند
- اگر امنیت قوی‌تر می‌خواهید و کمی کاهش throughput قابل قبول است، `2 — ChaCha20` را امتحان کنید

> **باید دقیقاً با تنظیم `DATA_ENCRYPTION_METHOD` سرور مطابقت داشته باشد.**

---

#### استراتژی بالانس (Balancing Strategy)
**گزینه‌ها:** `1 — تصادفی`، `2 — Round Robin`، `3 — کمترین افت`، `4 — کمترین تأخیر`
**پیش‌فرض:** `2 — Round Robin`
**بهینه برای ایران:** `3 — Least Loss (کمترین افت)`

کنترل می‌کند که کلاینت هر پکت را از کدام Resolver بفرستد.

- **تصادفی** — ساده‌ترین حالت، بدون بازخورد
- **Round Robin** — به‌طور یکنواخت بین همه Resolverها می‌چرخد
- **Least Loss** — Resolverهایی را که کمتر پکت drop می‌کنند ترجیح می‌دهد — **بهترین برای ایران** چون شبکه‌های ایران افت پکت بالا و نامتعادل دارند
- **Lowest Latency** — سریع‌ترین Resolver را ترجیح می‌دهد — اگر Resolverها خیلی یکنواخت هستند مناسب است

---

#### تکرار بسته (Packet Duplication)
**بازه:** `1–8`
**پیش‌فرض:** `2`
**بهینه برای ایران:** `2–3`

هر پکت را از این تعداد Resolver به صورت همزمان می‌فرستد. اگر یک مسیر پکت را drop کند، یک نسخه دیگر از مسیر دیگری می‌رسد. در شبکه پر افت ایران، `2` یک پایه محکم است. اگر قطعی‌های مکرر دارید، `3` را امتحان کنید. بالاتر از `4` نروید — وقتی Resolver زیاد دارید، پهنای باند هدر می‌رود.

---

#### حداقل MTU آپلود (Min Upload MTU)
**بازه:** `10–500`
**پیش‌فرض:** `38`
**بهینه برای ایران:** `38`

حداقل حجم payload آپلود (به بایت) که بعد از discovery پذیرفته می‌شود. Resolverهایی که فقط پکت‌های کوچک‌تر از این را می‌پذیرند حذف می‌شوند. `38` خیلی محافظه‌کارانه است — تقریباً هیچ Resolveری با این مقدار حذف نمی‌شود. برای ایران روی `38` بگذارید تا بیشترین تعداد Resolver قابل استفاده باشد.

---

#### حداکثر MTU آپلود (Max Upload MTU)
**بازه:** `10–500`
**پیش‌فرض:** `150`
**بهینه برای ایران:** `80–100`

حد بالای discovery برای MTU آپلود. MasterDnsVPN هر Resolver را probe می‌کند تا بزرگ‌ترین پکتی که می‌پذیرد را پیدا کند. در ایران، ترافیک DNS به شدت بررسی می‌شود — query های DNS بزرگ مشکوک به نظر می‌رسند و بیشتر drop می‌شوند. مقدار `80–100` به معنی استفاده از پکت‌های کوچک‌تر و کم‌تر مشکوک است که احتمال trigger کردن DPI را کاهش می‌دهد.

---

#### حداقل MTU دانلود (Min Download MTU)
**بازه:** `100–2000`
**پیش‌فرض:** `500`
**بهینه برای ایران:** `400`

حداقل حجم payload دانلود. Resolverهایی که پاسخ DNS خیلی کوچک برمی‌گردانند حذف می‌شوند. کاهش به `400` Resolverهای بیشتری در pool نگه می‌دارد — مفید در ایران که بسیاری از Resolverها کار می‌کنند اما پاسخ محافظه‌کارانه می‌دهند.

---

#### حداکثر MTU دانلود (Max Download MTU)
**بازه:** `100–2000`
**پیش‌فرض:** `900`
**بهینه برای ایران:** `700`

حد بالای discovery برای MTU دانلود. پاسخ‌های DNS بزرگ (۸۰۰-۹۰۰ بایت) احتمال بیشتری دارند که توسط ISPهای ایران fragmented شوند یا DPI را trigger کنند. مقدار `700` پکت‌های دانلود را در محدوده امن‌تری نگه می‌دارد که کمتر احتمال block شدن در حین انتقال دارند.

---

#### سطح لاگ (Log Level)
**گزینه‌ها:** `DEBUG`، `INFO`، `WARN`، `ERROR`
**پیش‌فرض:** `INFO`
**بهینه برای ایران:** `INFO`

کنترل می‌کند که MasterDnsVPN چقدر در ترمینال چاپ کند. `INFO` رویدادهای اتصال، تغییر Resolver و خطاها را نشان می‌دهد — به اندازه کافی برای اطلاع بدون پر کردن ترمینال. `DEBUG` را فقط برای عیب‌یابی مشکل خاصی استفاده کنید.

## 🖥️ بخش ۱ — راه‌اندازی سرور

### ۱.۱ 🌐 راه‌اندازی دامنه (پیش‌نیاز)

برای دریافت درخواست‌های DNS روی سرور، باید یک ساب‌دامین به آن delegate کنید.
**دو رکورد DNS** در کنترل پنل دامنه بسازید:

#### مرحله ۱.۱.۱ — ساخت رکورد A

| فیلد | مقدار |
|---|---|
| Type | `A` |
| Name | `ns` (یا هر اسم کوتاه) |
| Value | آدرس IPv4 سرور |

مثال: `ns.example.com → 1.2.3.4`

> **کاربران Cloudflare:** روی آیکون ابر نارنجی کلیک کنید تا **خاکستری (DNS only)** شود. نباید proxied باشد.

#### مرحله ۱.۱.۲ — ساخت رکورد NS

| فیلد | مقدار |
|---|---|
| Type | `NS` |
| Name | `v` (ساب‌دامین تانل) |
| Value | `ns.example.com` |

مثال: `v.example.com → ns.example.com`

> **کاربران Cloudflare:** رکورد NS را عادی اضافه کنید. مطمئن شوید رکورد A روی DNS only است.

#### ۱.۱.۳ 💡 نکته MTU

نام‌های کوتاه‌تر فضای بیشتری برای داده داخل هر پکت DNS باقی می‌گذارند.
نام‌ها را کوتاه نگه دارید (۱-۳ کاراکتر).

---

### ۱.۲ 🐧 نصب سرور لینوکس

#### مرحله ۱.۲.۱ — نصب خودکار

```bash
bash <(curl -Ls https://raw.githubusercontent.com/masterking32/MasterDnsVPN/main/server_linux_install.sh)
```

بعد از اتمام:
- سرور به صورت خودکار شروع می‌کند
- **کلید رمزنگاری** در ترمینال نشان داده می‌شود
- کلید در فایل `encrypt_key.txt` ذخیره می‌شود

> ⚠️ **کلید را ذخیره کنید** — کلاینت‌ها برای اتصال به آن نیاز دارند.

#### مرحله ۱.۲.۲ — نکات مهم

**دامنه:** همان ساب‌دامین رکورد NS را وارد کنید (مثلاً `v.example.com`).

**انتشار DNS:** تا ۴۸ ساعت صبر کنید.

**تأیید DNS:**
```bash
dig v.example.com NS
dig @ns.example.com v.example.com A
```

**باز کردن پورت ۵۳:**
```bash
# ufw
sudo ufw allow 53/udp && sudo ufw reload

# firewalld
sudo firewall-cmd --add-port=53/udp --permanent && sudo firewall-cmd --reload
```

**پورت ۵۳ در اشغال است (systemd-resolved):**
```bash
sudo systemctl stop systemd-resolved
sudo systemctl disable systemd-resolved
sudo systemctl restart MasterDnsVPN
```
